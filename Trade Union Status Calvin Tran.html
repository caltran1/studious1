Determining Trade Union Status¶

Calvin Tran¶

Project Goal:¶

Given data about union members, create a model that can predict whether additional members will remain in the union (or not).

Target variable: 'LeftUnion', a binary yes/no status.

Features: Certain demographic information (Gender, US Citizen, State of Residence, Married) and certain union speficic information (time in union, billing frequency, payment method).

Tools used:¶

This is a classification type problem with data that still needs to be cleaned and preprocessed. Additionally, comparing multiple models and then tuning them for best performance will require splitting into a cross-validation set along with some data visualization.

This is well suited to using:

  • Python pandas and numpy to view the data and work with it in dataframe and array formats
  • sklearn implementation of different classification models, in addition to preprocessing, metrics, and splitting into train/validation/test sets.
  • the matplotlib and seaborn libraries to visualize feature distribution and correlation.

3. Separate features and exploratory analysis¶

Visual presentation choices:¶

Numeric data:¶

Best fits a histogram. This chart type shows the distribution with 5-number quartiles and shows any outliers which can then be further adressed.

Numeric features can also be visualized together as a pairplot. There is good reason from exploratory data analysis to believe that some of the features are collinear. Numeric features can also calculate their correlation coefficient directly.

By using a corrplot with heatmap-style coloring, it will be easy to visualize strong collinearity for further data preprocessing,.

Categorical data:¶

Best suits a bar chart. While univariate analysis is just a simple value count, by doing a bivariate bar chart with the target variable, we can instead visualize how each category affects the target by their relative sizes to each other.

Why is this especially useful? Because if the different category options have the same target variable distribution, then it is clear that that feature has no effect on the classification.

Similarly, if different features happen to share the same distribution as each other, then those features represent the same effect and they are collinear.

Overview of Preliminary Models:¶

  • Only ~75-80% accuracy on the validation set which is not very good.
  • How this be improved?

Most models are fitting training data very well, but accuracy drops a lot for the validation set. In contrast, the LR model is unlikely to overfit and has a lower accuracy on the test set as well. This is a case of overfitting.

One strategy to combat overfitting is to reduce model complexity so that it can effectively generalize to new data

  • Explore options for feature selection and feature engineering

  • regularize linear model

  • tune hyperparameters for other models

Feature Selection & Reprocessing¶

Which features were already chosen to be left out?¶

  • For numeric features, 'MonthsInUnion' and 'TotalDues' were collinear. This was determined by computing their correlation coefficient, in addition to simply viewing their pairplot to be linear. As expected, the more time someone is in the union, the higher their total dues.

Which categorical features are irrelevant?¶

features that have little to no difference on the yes/no distribution regardless of variable response.

  • the yes/no distribution with gender (male/female) and state of residence (MO/IL) are the same. These features are not very informative.
  • 'USACitizen' and 'ContinuingEd' also look close.

Features with the same distribution as each other, they represent the same thing

  • Feature B, C, D, E, F have similar distributions, so I will only keep feature F

Overall the list of categorical features has been reduced by 8 compared to the previous set of models.

Reducing the number of features is a good strategy to prevent overfitting (as shown with the training data performance being notably higher than the validation set performance), in addition to having the model train on the feature that most directly contribute to predicting the target variable rather than noise.

Is this dataset a good candidate for PCA?¶

  • Overall the dataset does not contain many training examples or features.
  • PCA results in a loss of information and explanability for the model. I would instead first bet on the more manual approach to feature selection seen above as it maintains the explanability of the model while still having a workable number of features.
  • As a result, PCA seems better suited as a future consideration or alternate approach if the model tuning calls for it.

Model Write-up & Comments on model selection¶

As mentioned above, this is a binary classification type problem. A lot of popular models exist for this type of problem:

  • Logistic Regression: Sigmoidal classifier that is good for a problem like this one where there are only 2 classes.
  • Decision Trees: Uses features for attribute tests that determine the final output.
  • Random Forests: Similar to decision trees but ensemble model that is more robust
  • Neural Networks: Transforms inputs with weights and activation functions for output.
  • Support Vector Machine: Classifier that finds the optimal decision boundary between the two classes.

Since these all work for binary classification problems, and they all have sklearn implementations, I will choose to implement all five of these.

With the cross-validation set that was split earlier, these models can be compared and the one with highest performance selected to be run on the test set.

Overview on second round of models, summary of results and next steps¶

These models are now underfitting the training data compared to the first set of models. However their performance on the validation set is relatively unchanged.

One strategy is to try some feature engineering with the features that were kept. Since the model is not overfitting, if performance on the training set goes up, the validation set should follow.

Changes can be easily verified by simple logistic regression.

Recall that earlier I checked "what proportion of the data left the union?"

A baseline model that assumes nobody ever leaves the union (ie. doesn't do anything put output 0) itself has a performance of 1495/1995, or 0.749.

For 3 of the models, model performance on the validation set was around 0.83. The provided features of course provide a better prediction than simply guessing "no" for each individual.

What cutoff seems appropriate as good enough? As mentioned above I do think there is a little more that can be done in terms of feature engineering to try and coax out a little better performance.

However, it could very well be that the provided features are simply not enough to provide better predictions. Broadly, the dataset only has features for basic demographics and union payments.

Ideally more informative features would have included things such as:

  • Previous union engagement
  • Some metric of job satisfaction
  • Job position beyond management/non-management

In particular these features seem really ripe for binning the data and that may lead to better performance.
